# Story 5.1: Pydantic Data Boundaries

Status: in-progress

## Story

As a Developer,
I want all core data structures (AttackResult, ModuleMetadata) to be Pydantic models,
So that I can easily serialize them to JSON, validate them at runtime, and ensure a clean "data boundary" between the engine and the UI.

## Acceptance Criteria

1. **Given** an `AttackResult` is generated by the engine
2. **When** I call `.model_dump_json()` on it
3. **Then** it produces a valid JSON string containing all nested fields (Score, Metadata, etc.)
4. **And** all core classes in `src/core/engine.py` inherit from `pydantic.BaseModel`.
