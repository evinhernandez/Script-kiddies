# BMAD + Project Memory Integration
We are using the official BMAD (Build More Architect Dreams) Method (installed in `_bmad/`) in conjunction with this `project-memory/` directory (inspired by the SkillsLLM Marketplace).

## AI Agent Workflow:
1.  **Follow BMAD Workflows:** When asked to perform a role (Analyst, PM, Architect), refer to the `.claude/` or `_bmad/` prompt templates generated by the BMAD installer.
2.  **State & Memory:** Instead of relying purely on BMAD's default output folders, persist high-level architecture decisions, sprints, and rules in THIS `project-memory/` directory.
    - `rules.json`: Our core coding standard.
    - `active_context.md`: The immediate sprint goals.
    - `architecture_map.xml`: The source of truth for repository structure.
3.  **Cross-pollination:** When running `/bmad-help` or any BMAD task, always ingest `project-memory/active_context.md` first to understand the current state.
